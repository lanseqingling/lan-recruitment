# 一、项目背景与目标

## 1.1 项目背景

随着互联网招聘平台的发展，传统招聘流程中岗位与简历匹配效率低、人工筛选成本高的问题日益突出。为提升招聘效率，本系统引入基于余弦相似度算法的智能匹配机制，实现岗位与简历之间的自动化匹配与推荐。

## 1.2 项目目标

- 本项目旨在设计并实现一个面向求职者、企业 HR 和管理员的智能招聘系统，完成以下目标：
- 为求职者提供在线简历管理、岗位浏览与智能推荐功能；
- 为 HR 提供岗位发布、简历筛选与智能推荐支持；
- 基于余弦相似度算法，实现岗位与简历的匹配度计算与排序；
- 构建结构清晰、功能完整、适合本科毕业设计展示的招聘系统。

# 二、系统角色与权限划分

系统采用基于角色的功能划分模式，不同角色在统一页面结构下拥有不同的功能入口。

## 2.1 求职者（USER）

主要面向普通用户，核心职责为求职与简历管理：
- 注册与登录系统
- 创建和维护个人在线简历
- 选择简历相关标签（技能、行业、城市等）
- 浏览与搜索岗位信息
- 接收系统推荐岗位
- 向目标岗位投递简历

## 2.2 企业 HR（HR）

主要面向招聘方用户，核心职责为岗位管理与简历筛选：
- 注册与登录系统
- 发布与维护招聘岗位
- 为岗位配置标签信息
- 查看投递到岗位的简历
- 基于系统推荐结果筛选简历

## 2.3 系统管理员（ADMIN）

负责系统整体维护与管理：
- 审核 HR 账号与岗位信息
- 维护系统标签库
- 管理系统基础配置与权限
- 发布系统公告

> 管理员账号由系统后台统一配置，不提供独立注册入口。

# 三、系统总体功能结构

系统功能按照通用功能模块 + 角色功能模块 + 智能推荐模块进行划分，结构清晰，便于论文描述与实现。

# 四、通用功能模块

## 4.1 用户注册与登录

- 支持邮箱 + 验证码登录
- 支持用户名 + 密码登录
- 注册信息包括：用户名、密码、邮箱
- 登录时根据用户角色展示对应功能入口

## 4.2 个人中心

- 修改个人基本信息
- 修改密码
- 上传或更换头像

# 五、求职者功能模块

## 5.1 简历管理

### 5.1.1 在线简历维护

求职者可创建并编辑在线简历，主要内容包括：
- 基本信息：姓名、学历、城市
- 求职意向：期望岗位类型、期望薪资
- 工作与项目经历

### 5.1.2 简历标签配置

为便于智能匹配，简历需绑定标签信息：
- 技能类标签（需设置熟练度：精通 / 熟悉 / 了解）
- 行业、经验等标签（无需熟练度）

### 5.1.3 简历文件上传

支持上传附件简历（PDF / JPG / PNG），作为在线简历的补充。

## 5.2 岗位浏览与投递

- 支持按关键词、城市、岗位类型等条件搜索岗位
- 系统首页展示推荐岗位列表
- 查看岗位详情并进行简历投递
- 查看个人投递记录与投递状态

# 六、HR 功能模块

## 6.1 岗位管理

### 6.1.1 岗位发布与编辑

HR 可发布招聘岗位，岗位信息包括：
- 岗位名称
- 薪资范围
- 工作城市
- 岗位类型
- 岗位描述

### 6.1.2 岗位标签配置

为岗位绑定标签，用于后续智能匹配：
- 技能类标签（设置需求熟练度）
- 行业、经验类标签

## 6.2 简历筛选

- 查看投递到岗位的简历列表
- 系统根据匹配度对简历进行排序推荐
- 支持按学历、标签等条件进行人工筛选

# 七、管理员功能模块

## 7.1 标签库管理

- 维护系统统一标签库
- 包括技能标签、岗位类型、行业标签等

## 7.2 内容与账号审核

- 审核 HR 注册申请
- 审核岗位发布信息

## 7.3 系统配置与公告

- 管理系统基础参数
- 发布系统公告信息

# 八、智能推荐与匹配模块（核心）

## 8.1 算法设计思想

- 系统核心采用余弦相似度算法，用于计算：
- 简历与岗位之间的匹配度
- 匹配结果用于推荐与排序，不直接做自动决策

## 8.2 特征向量构建

- 将简历与岗位统一表示为「标签 - 权重」向量
- 标签来自系统标签库
- 不同类型标签采用不同权重规则

权重规则说明：
- 技能类标签：
    - 基础权重：3
    - 熟练度系数：精通3 | 熟悉2 | 了解1
    - 最终权重 = 基础权重 × 熟练度系数
- 行业 / 经验类标签：
    - 基础权重：1
    - 无熟练度系数

## 8.3 相似度计算与推荐逻辑

- 使用余弦相似度公式计算岗位向量与简历向量之间的相似度
- 按相似度值进行降序排序
- 排序结果用于：
    - 求职者首页岗位推荐
    - HR 简历筛选排序
