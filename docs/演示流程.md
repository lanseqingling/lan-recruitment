# 智能招聘系统（演示流程）

## 启动

- 后端
  - 进入 backend 目录执行：`mvn -DskipTests spring-boot:run`
  - 健康检查：`GET http://localhost:8080/api/health`
- 前端
  - 进入 frontend 目录执行：`npm install && npm run dev`
  - 访问：`http://localhost:5173/`

## 内置演示账号

- 管理员：admin / admin123456
- 求职者：user1 / user123456
- HR：hr1 / hr123456

说明：内置账号与默认标签由 [BootstrapDataRunner](file:///Users/bytedance/java/lan-recruitment/backend/src/main/java/com/lanrecruitment/config/BootstrapDataRunner.java) 初始化。

## 核心流程（推荐走前端页面）

### 0) 通用：个人中心上传头像

- 登录任意账号
- 顶部导航栏右侧点击「头像 + 用户名」下拉
- 进入「个人中心」→ 点击「上传/更换头像」

对应接口：
- `POST /api/common/user/avatar/upload`（multipart: file）
- `GET /api/common/user/profile`

### 1) 管理员：审核 HR

- 登录管理员账号
- 登录页默认是「用户名密码登录」
- 进入「审核管理」
- 在「HR审核」tab 对 hr1 点击「通过」

对应接口：
- `GET /api/admin/audit/hr/list`
- `POST /api/admin/audit/hr/decision`（id + pass）

### 2) HR：发布岗位 + 配置标签 + 提交审核

- 登录 HR 账号
- 进入「岗位管理」→「发布岗位」
- 保存后岗位进入「待审核」
- 点击「标签」配置岗位标签（技能可选熟练度）

对应接口：
- `POST /api/hr/job/save`
- `POST /api/hr/job/tags/save`

### 3) 管理员：审核岗位

- 切回管理员账号
- 进入「审核管理」→「岗位审核」
- 对刚才 HR 提交的岗位点击「通过」

对应接口：
- `GET /api/admin/audit/job/list`
- `POST /api/admin/audit/job/decision`（id + pass）

### 4) 求职者：创建简历 + 配置简历标签

- 登录求职者账号
- 进入「简历管理」→「新增简历」→ 保存
- 点击「标签」配置简历标签（技能可选熟练度）

对应接口：
- `POST /api/user/resume/save`
- `POST /api/user/resume/tags/save`

### 5) 求职者：首页查看推荐 + 投递

- 进入「首页」
- 顶部「推荐岗位」展示余弦相似度匹配度（matchScore）
- 点击「投递」即可对默认简历投递

对应接口：
- `GET /api/user/job/recommend`
- `POST /api/user/job/apply`

### 6) HR：查看投递简历 + 按匹配度排序筛选

- 登录 HR 账号
- 进入「岗位管理」→ 某岗位点击「投递」
- 列表会返回 matchScore 并按得分降序排序
- 可标记「已看」/「通过」/「拒绝」

对应接口：
- `GET /api/hr/job/applications`
- `POST /api/hr/job/apply/status`

## 关键点

- 推荐与筛选使用「简历标签向量 × 岗位标签向量」余弦相似度；结果落库到 `match_score`，再次查询会更新分数与时间。
- 岗位在「招聘中 + 审核通过」后才会进入公开列表与推荐列表。
