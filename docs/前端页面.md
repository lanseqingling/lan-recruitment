# 一、整体风格定位（强约束）
- 本系统为本科毕业设计项目
- 页面风格要求：
    - 现代但克制
    - 偏系统型 / 教学型
    - ❌ 不要像成熟商业招聘平台
    - ❌ 不要过度精致、复杂或炫技
- 目标效果：导师一看就知道是完整、规范、逻辑清晰的毕业设计系统

# 二、配色与视觉风格
- 主色调：蓝青色
- 背景色：白色 / 浅灰色
- 整体风格偏理性、技术感
- ❌ 避免大面积渐变
- ❌ 避免高饱和对比色
- ❌ 避免“可爱风 / 移动端风”
- 圆角：偏小圆角或直角，不要过于圆润

# 整体布局结构（所有角色共用）
- 统一系统外观，不为不同角色设计不同 UI
- 所有角色共用一套页面结构，仅功能和入口不同

通用页面布局：
- 顶部导航栏（固定）
    - 左侧：系统名称 / Logo
    - 中间：搜索框（岗位 / 内容搜索）
    - 右侧：消息入口、简历 / 个人入口、用户头像
- 主体区域：
    - 左侧：功能导航栏 / 筛选区
    - 右侧：主内容区域
      登录 / 注册页设计：
- 登录页风格参考 Boss 直聘
- 整体风格：
    - 灰白色背景
    - 页面居中卡片布局
    - 简洁、克制、偏系统型
- 卡片内自上而下结构：
    1. 系统名称 / 简要说明
    2. 登录方式切换区域（「我要找工作」或「我要招聘」，管理员（ADMIN）无需特殊登录入口）
    4. 登录表单（只有输入框和Placeholder Text）
    5. 登录 / 注册按钮（和输入框等长）
    6. 角色入口切换（找工作 / 招聘）
- 登陆方式：
    1. 邮箱 + 验证码登录
    2. 用户名 + 密码登录
- 注册方式：用户名 + 密码 + 邮箱 + 验证码

# 四、首页与核心页面结构约定

- 首页为岗位浏览与推荐页面
- 页面采用左右结构：
    - 左侧：固定筛选卡片（条件筛选）
    - 右侧：岗位卡片列表或岗位列表 + 详情区域
- 岗位以卡片形式展示
- 页面采用 Feed 流
    - ❌ 不使用分页组件
    - ✅ 可使用“加载更多”或简单下拉加载

# 五、角色差异展示原则（不展开细节）

系统包含：
- 求职者（USER）
- HR
- 管理员（ADMIN）

前端展示原则：
- ❌ 不拆多套页面结构
- ❌ 不复杂化角色切换逻辑
- ✅ 根据角色：
    - 控制菜单项显示
    - 控制按钮和功能入口
- 页面逻辑清晰，便于论文中说明「基于角色的功能区分」

# 六、组件与实现风格约束（重要）

- 技术栈：Vue 3 + Vite + Element Plus
- ❌ 避免大量原生 JS / DOM 操作
- ❌ 避免复杂 Vue 高级写法
- ❌ 避免过度组件拆分
- ✅ 页面级开发为主
- 不追求组件高度复用，以易读、好维护为第一目标
- 样式优先使用 Element Plus，少量自定义 CSS 即可

# 七、与论文主题的风格呼应（隐性要求）

- 页面设计需能自然承载：
    - 岗位推荐
    - 匹配度展示
    - 不同角色操作入口
- 不要求复杂算法可视化
- 页面重点是**“结果展示 + 功能完整”**，而非算法表现形式

# 八、给 AI 的最终提醒（防翻车）

在生成前端页面时：
- 宁可简单，也不要复杂
- 宁可像毕业设计，也不要像商业产品
- 页面结构应方便：
    - 截图
    - 写论文中的「系统界面设计」
    - 向导师讲清楚系统逻辑
